0. Bàn giao nhanh cho dev mới

Việc nên làm trong ngày đầu

  1. Clone/addon path: đảm bảo thư mục module l10n_vn_b03dn_direct_report nằm trong --addons-path, chạy cập nhật apps list và module đã ở trạng thái Installed trên DB dev.
  2. Đọc __manifest__.py: tên module người dùng thấy, depends (đặc biệt accounting_adecsol, account_financial_report), và danh sách data/ + views/ được load.
  3. Mở UI: Accounting → Configuration → B03-DN templates (menu menu_l10n_vn_b03dn_template, parent account.menu_finance_configuration). Xác nhận form khớp với field trong models/b03dn_template.py.
  4. Chạy test (từ root addons chứa module):
    odoo-bin -c odoo.conf -d YOUR_DATABASE \
  --test-enable --stop-after-init -i l10n_vn_b03dn_direct_report
    hoặc chỉ chạy file: tùy setup CI — file gốc: tests/test_b03dn_direct.py.

1. Phạm vi

Báo cáo lưu chuyển tiền tệ B03-DN phương pháp trực tiếp lấy quy tắc từ template (không hard-code chỉ tiêu trong Python). Tài liệu này mô tả field, ràng buộc, XML mẫu và chỗ mở rộng trong code.

4. Trường dòng (l10n.vn.b03dn.line)

5. line_kind — hành vi engine

Giá trịNguồn số
noneLuôn 0; có thể không có code (tiêu đề phần).
leafCộng các fragment khớp; aml_ids = id các dòng tiền.
aggregateCộng các mã trong sum_expression; gộp aml_ids.
opening_cashSUM(debit - credit) TK tiền, date < date_from, posted.
fx_adjustmentTổng balance AML trong kỳ, TK khớp fx_account_patterns.

Vòng aggregate có thể lặp cho đến khi biểu thức ổn định (phụ thuộc các mã con đã có số).

6. Pattern tài khoản

  • Phân tách bằng dấu phẩy, khoảng trắng sau phẩy được trim khi parse.
  • Suffix % → tiền tố: mã TK đối ứng phải bắt đầu bằng phần trước %.
  • Không có % → khớp đúng mã.
  • Tiền vào: engine chỉ dùng credit_account_patterns cho fragment có is_inflow=True.
  • Tiền ra: chỉ dùng debit_account_patterns khi is_inflow=False.

XML mẫu TT200 đôi khi ghi cả hai nhánh pattern trên cùng leaf để đối chiếu tài liệu; nhánh không dùng cho chiều hiện tại không ảnh hưởng kết quả.

7. Ràng buộc & lỗi thường gặp

  • _b03dn_line_code_required_by_kind: mọi dòng trừ none phải có code non-empty sau strip.
  • _b03dn_line_code_unique_per_template: không trùng code trong cùng template_id.
  • Biểu thức sum_expression: mọi token mã tham chiếu phải tồn tại ở dòng khác cùng template (nếu không engine bỏ qua phần chưa resolve).

8. Dữ liệu XML mặc định

  • data/b03dn_template_tt200_direct.xml — template «B03-DN — Phương pháp trực tiếp (TT200)» và toàn bộ dòng mã 01…70.
  • data/account_tags_cash_flow.xml — thẻ phục vụ exclude/include (vd loại một số TK khỏi leaf cụ thể).

Khi nâng cấp module: dùng migrations/18.0.x.x/pre-migrate.py (theo version đang nâng) để đổi tên field / dữ liệu; đối chiếu số phiên bản trong __manifest__.py.

9. Engine — tham chiếu code

File models/b03dn_engine.py: compute_period, _fragment_from_cash_line, phân bổ multi-counterpart, _code_matches_patterns, match leaf, merge aggregate.

Chi tiết luồng fragment / ví dụ số: lctt_tt200_calculation_technical.html.

10. Mở rộng
